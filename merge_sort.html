🔷 合併排序法

Merge Sort - 分治法的經典排序演算法

📌 什麼是合併排序法？

合併排序法（Merge Sort）是一種基於「分治法（Divide and Conquer）」的排序演算法。 它會先將數列不斷分割成小部分，再將排序後的小部分逐步合併，最終得到完整排序結果。

🎯 演算法原理

  1. 將數列不斷對半分割
  2. 直到每個子陣列只剩一個元素
  3. 將子陣列兩兩合併並排序
  4. 重複合併直到整個數列完成排序

💻 代碼示例（C）

#include <stdio.h> void merge(int arr[], int l, int m, int r) { int i = l, j = m + 1, k = 0; int temp[100]; while(i <= m && j <= r) { if(arr[i] <= arr[j]) temp[k++] = arr[i++]; else temp[k++] = arr[j++]; } while(i <= m) temp[k++] = arr[i++]; while(j <= r) temp[k++] = arr[j++]; for(i = l, k = 0; i <= r; i++, k++) arr[i] = temp[k]; } void mergeSort(int arr[], int l, int r) { if(l < r) { int m = (l + r) / 2; mergeSort(arr, l, m); mergeSort(arr, m + 1, r); merge(arr, l, m, r); } } int main() { int arr[] = {38, 27, 43, 3, 9, 82, 10}; int n = 7; mergeSort(arr, 0, n - 1); printf("排序後: "); for(int i = 0; i < n; i++) printf("%d ", arr[i]); return 0; }

📊 複雜度分析

項目複雜度說明
時間複雜度（最佳）O(n log n)每次都需要分割與合併
時間複雜度（平均）O(n log n)穩定表現
時間複雜度（最差）O(n log n)不受資料影響
空間複雜度O(n)需要額外陣列
穩定性穩定相同元素順序不變

✅ 優點與缺點

✓ 優點

  • 時間複雜度穩定 O(n log n)
  • 適合大數據排序
  • 穩定排序
  • 適合鏈結串列

✗ 缺點

  • 需要額外記憶體
  • 實作較複雜
  • 遞迴可能影響效能

🔍 執行範例

原始陣列：[38, 27, 43, 3]

分割 → [38,27] [43,3]

再分 → [38][27] [43][3]

合併 → [27,38] [3,43]

最終 → [3,27,38,43]
