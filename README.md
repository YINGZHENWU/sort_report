## 一、前言
排序（Sorting）是電腦科學中最基本且重要的演算法之一，其目的是將一組資料依照特定順序排列，例如由小到大或由大到小。當資料完成排序後，搜尋與資料分析會變得更加有效率，因此排序演算法被廣泛應用在資料庫、搜尋系統以及各種程式設計中 。
在學習演算法的過程中，不同排序方法的效率差異是一個重要的研究主題。有些排序方法雖然概念簡單，但在資料量變大時效率會明顯下降；而有些演算法則能在大量資料下維持較好的效能。因此，本報告將介紹五種常見的排序演算法，並透過程式實驗比較其排序效率，進一步了解不同演算法在實際應用中的表現。

---

## 二、排序法原理

### 1. 氣泡排序（Bubble Sort）
氣泡排序是一種簡單的排序方法，其原理是不斷比較相鄰兩個元素，如果順序錯誤就交換位置，使較大的數逐漸移動到陣列尾端。

**操作方式**
1. 從陣列開頭開始比較相鄰元素  
2. 若前者大於後者則交換  
3. 每一輪會將最大值移到最後  
4. 重複直到排序完成  

---

### 2. 選擇排序（Selection Sort）
選擇排序的概念是每次從未排序資料中找出最小值，並與目前位置的元素交換。

**操作方式**
1. 從未排序區找出最小值  
2. 與當前位置元素交換  
3. 排序區逐漸增加  
4. 重複直到排序完成  

---

### 3. 插入排序（Insertion Sort）
插入排序的概念類似整理撲克牌，每次將一個元素插入到已排序序列中的適當位置。

**操作方式**
1. 取出下一個元素  
2. 與已排序元素比較  
3. 找到正確位置插入  

---

### 4. 合併排序（Merge Sort）
合併排序使用分治法，將陣列不斷分割成較小的子陣列，排序後再將它們合併。

---

### 5. 快速排序（Quick Sort）
快速排序透過選擇一個基準值（pivot），將小於基準值的元素放在左側，大於基準值的元素放在右側，再分別進行排序。

---

## 三、複雜度分析

| 排序法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 |
|------|------|------|------|
| Bubble Sort | O(n²) | O(n²) | O(1) |
| Selection Sort | O(n²) | O(n²) | O(1) |
| Insertion Sort | O(n²) | O(n²) | O(1) |
| Merge Sort | O(n log n) | O(n log n) | O(n) |
| Quick Sort | O(n log n) | O(n²) | O(log n) |

其中前三種排序法時間複雜度較高，因此在大量資料時效率較差，而後兩種排序法效率較高。

## 四、實驗設計（數據模擬）

為了比較不同排序演算法的效率，本實驗使用程式產生隨機整數並進行排序。

**實驗設定**
- 使用亂數產生整數資料
- 測試不同資料量

| 測試次數 | 資料量 n |
|------|------|
| 測試1 | 1000 |
| 測試2 | 5000 |
| 測試3 | 10000 |
| 測試4 | 50000 |

**實驗流程**
1. 產生 n 個隨機整數  
2. 使用五種排序演算法進行排序  
3. 記錄每種排序所花費時間  
4. 重複測試並比較結果

## 六、心得與結論

透過這次排序演算法的分析與實驗，我更了解不同演算法在效率上的差異。像氣泡排序、選擇排序和插入排序雖然概念簡單，但在資料量變大時排序速度會明顯下降，因此較適合用在小型資料或教學用途。

相比之下，合併排序與快速排序利用分治法的概念，使時間複雜度降低到 O(n log n)，在大量資料排序時表現較佳。在本次實驗中也可以明顯看到，當資料量增加時，這兩種排序方法仍能保持較快的速度。

透過這次報告，我不僅學習到排序演算法的基本原理，也了解到演算法效率對程式效能的重要性。未來在設計程式時，選擇合適的演算法可以讓程式運行更有效率，這也是學習演算法的重要原因。
