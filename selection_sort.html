🔹 選擇排序法

Selection Sort - 一種直觀的排序演算法

📌 什麼是選擇排序法？

選擇排序法（Selection Sort）是一種簡單直觀的排序演算法。它每次從未排序部分選擇最小（或最大）元素，然後將其放到已排序部分的末端，直到整個數列排序完成。

🎯 演算法原理

  1. 從未排序部分找出最小值
  2. 將最小值與未排序部分的第一個元素交換
  3. 將已排序部分的長度增加 1，重複以上步驟直到排序完成

💻 代碼示例

C 實現：

#include <stdio.h> void selection_sort(int arr[], int n) { for(int i = 0; i < n-1; i++) { int min_idx = i; for(int j = i+1; j < n; j++) { if(arr[j] < arr[min_idx]) { min_idx = j; } } int temp = arr[i]; arr[i] = arr[min_idx]; arr[min_idx] = temp; } } int main() { int arr[] = {64, 25, 12, 22, 11}; int n = 5; selection_sort(arr, n); printf("排序後的陣列: "); for(int i=0; i<n; i++) printf("%d ", arr[i]); return 0; }

📊 複雜度分析

指標 複雜度 說明
時間複雜度（最差/平均） O(n²) 需要比較 n(n-1)/2 次
時間複雜度（最好） O(n²) 即使已排序，也需要遍歷 n(n-1)/2 次
空間複雜度 O(1) 只需要常數額外空間
穩定性 不穩定 相等元素的相對位置可能改變

✅ 優點與缺點

✓ 優點

  • 概念簡單直觀
  • 實現容易
  • 不需要額外空間
  • 適合小型數據排序

✗ 缺點

  • 時間複雜度高 O(n²)
  • 效率低於快速排序、歸併排序等
  • 不穩定
  • 大數據集不適用

🔍 執行範例過程

初始陣列： [64, 25, 12, 22, 11]

第一次遍歷： 找到最小值 11 → 與第一個元素交換 → [11, 25, 12, 22, 64]

第二次遍歷： 找到最小值 12 → 與第二個元素交換 → [11, 12, 25, 22, 64]

第三次遍歷： 找到最小值 22 → 與第三個元素交換 → [11, 12, 22, 25, 64]

第四次遍歷： 找到最小值 25 → 與第四個元素交換 → [11, 12, 22, 25, 64]

排序完成： [11, 12, 22, 25, 64]

💬 重點總結：選擇排序法簡單易懂，但效率低下且不穩定。適合學習排序概念或小規模數據，對大數據集不建議使用。
